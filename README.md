# ViaVpn

Via — это не полноценная коммерческая Vpn-платформа, а практичный Telegram-обвязочный сервис для управления Vpn-доступом через панель 3x-ui. Он помогает автоматизировать базовые операции: создавать пользователей, выдавать ссылки, генерировать QR-коды, организовывать поддержку и выполнять простые админские задачи. Для больших сетей, высокой нагрузки и серьёзной production-эксплуатации он подходит лишь частично.

## Назначение проекта

Проект рассчитан на небольшие и средние сценарии использования. В реальности он может стабильно работать на уровне примерно 100–200 пользователей, но это зависит от:

- мощности VPS;
- количества одновременно активных соединений;
- качества настроек 3x-ui;
- нагрузки на API и Telegram-ботов.

Если у вас большой трафик, много пользователей и высокая конкуренция за соединения, лучше рассматривать более полноценную инфраструктуру с мониторингом, отдельными узлами, балансировкой и более сложной архитектурой.

## Что умеет проект

- создавать VPN-пользователей через Telegram;
- выдавать ссылки для подключения;
- генерировать QR-коды;
- работать с базовыми инвайтами и рефералами;
- вести простую поддержку через отдельный бот;
- хранить данные локально в SQLite.

## Ограничения проекта

Важно понимать, что этот сервис не является полноценной production-системой в классическом смысле. У него есть ограничения:

- он не рассчитан на очень высокую нагрузку;
- у него нет продвинутой панели управления, аналитики, автоматического масштабирования и полноценного мониторинга;
- базовые функции бота реализованы достаточно просто;
- данные хранятся локально в SQLite, что подходит для небольшого проекта, но не для больших enterprise-сценариев.

## 1. Покупка и подключение VPS

Для начала нужно арендовать сервер, на котором будет работать VPN. Выберите любого удобного хостера, но обратите внимание на следующие пункты:

- публичный IP в тарифе;
- возможность смены IP;
- достаточную мощность для будущей нагрузки.

Я использую сервис aeza, но подойдет любой другой хостер с подходящими условиями.

При покупке сервера выберите Ubuntu последней версии или Debian — для нашего проекта разница фактически не критична. После покупки вы получите IP, логин (обычно `root`) и пароль.

Подключитесь к серверу по SSH:

```bash
ssh root@YOUR_SERVER_IP
```

## 2. Домен

Домен нужен, чтобы получить удобный адрес вместо IP-адреса. Например, `https://max.ru` выглядит лучше, чем `https://192.168.0.1`.

Купите домен у любого регистратора и не потеряйте доступ к учётной записи. Затем в настройках DNS создайте **две** записи типа `A`, обе указывающие на IP вашего сервера:

- имя: `@`, тип: `A`, значение: ваш IP, TTL: по умолчанию — основной домен (например, `my-via.ru`);
- имя: `sub`, тип: `A`, значение: ваш IP, TTL: по умолчанию — поддомен для страницы подписки (например, `sub.my-via.ru`).

Поддомен `sub.ВАШ_ДОМЕН` используется веб-страницей, на которую пользователи попадают по ссылке подписки (импорт конфига в приложение), — отдельно от главной страницы сайта. Обе DNS-записи должны быть созданы **до** выпуска SSL-сертификатов в следующем шаге, иначе проверка домена не пройдёт.

## 3. Установка Docker

На сервере установим Docker, через который будут запускаться Telegram-боты.

```bash
apt update && apt install curl -y
```

```bash
curl -fsSL https://get.docker.com | sh
```

## 4. Установка 3x-ui

3x-ui — это панель управления, с помощью которой вы будете создавать пользователей, настраивать протоколы, генерировать конфиги и смотреть статистику. Без неё бот не сможет управлять VPN.

### Установка

```bash
bash <(curl -Ls https://raw.githubusercontent.com/mhsanaei/3x-ui/master/install.sh)
```

Во время установки:

- выберите базу данных `sqlite`;
- укажите порт панели или оставьте случайный;
- выберите использование домена и укажите ваш домен.

После установки вы получите URL, логин и пароль для панели — сохраните эти данные.

### Рекомендуемые настройки 3x-ui

- включите BBR для улучшения сетевой производительности;
- обновите геоданные (например, `Update geo files`);
- установите и настройте `firewall`;
- по возможности смените порт SSH и настройте доступ по ключам.

Откройте панель:

```text
https://DOMAIN:YOUR_PORT/WEB_PATCH
```

### Выпуск SSL-сертификатов (основной домен + sub)

Веб-страница сайта работает на двух именах: основном домене (главная страница) и поддомене `sub.ВАШ_ДОМЕН` (страница подписки). Для HTTPS нужен отдельный сертификат под каждое имя — 3x-ui умеет выпускать их через встроенный acme.sh.

В консоли сервера запустите:

```bash
x-ui
```

В меню выберите пункт управления SSL-сертификатами и выпустите сертификат сначала для основного домена, затем повторите для поддомена `sub.ВАШ_ДОМЕН` (например, `my-via.ru` и `sub.my-via.ru`). Убедитесь, что обе A-записи из шага 2 уже применились (DNS может обновляться до нескольких часов) — иначе проверка домена не пройдёт.

После успешного выпуска вы увидите пути к файлам вида:

```text
Domain: my-via.ru
        Certificate Path: /root/cert/my-via.ru/fullchain.pem
        Private Key Path: /root/cert/my-via.ru/privkey.pem
Domain: sub.my-via.ru
        Certificate Path: /root/cert/sub.my-via.ru/fullchain.pem
        Private Key Path: /root/cert/sub.my-via.ru/privkey.pem
```

Ничего вручную переносить не нужно — `docker-compose.yml` уже монтирует `/root/cert` внутрь контейнера `web` и сам подхватывает нужные сертификаты по имени домена из `qq.env`.

### Настройка панели 3x-ui для работы с ботом

Для работы бота нужны:

- `XUI_URL` — URL панели, например `https://DOMAIN:YOUR_PORT/WEB_PATCH`;
- `XUI_API_KEY` — API-ключ панели.

Перейдите во вкладку "Входящие" и создайте нужные протоколы. Рекомендуется создать несколько вариантов, например `vless`, `vmess`, `trojan`, `hysteria`, и использовать разные типы транспорта: `websocket`, `grpc`.

Для входящих настроек важно указать:

- `sockopt`;
- `TCP Keep Alive Idle = 60`;
- `TCP Max Seg` `1420`. - Эта настройка индивидуальна и может только все сломать, на ваш страх и риск!

В разделе безопасности выберите `TLS` или `Reality`, в зависимости от доступных опций. Для `TLS` укажите домен в `SNI` и установите сертификат панели. Для `Reality` найдите цель, вставьте её и запустите проверку.

Затем перейдите в раздел `Учётные записи` и создайте API-токен. Он и будет `XUI_API_KEY`.

## 5. Настройка Telegram-ботов

Осталось подключить Telegram-ботов.

Создайте два бота через BotFather.

### Основной бот

1. Откройте BotFather.
2. Создайте нового бота.
3. Получите токен.
4. Сохраните его в `qq.env` как `BOT_TOKEN`.

### Бот поддержки

1. Создайте ещё одного бота.
2. Получите токен.
3. Сохраните его в `qq.env` как `SUPPORT_BOT_TOKEN`.

### Получение Telegram ID

Чтобы узнать свой Telegram ID, используйте @userinfobot или отправьте сообщение боту.

Укажите полученный ID в `ADMIN_ID` и `SUPPORT_ADMIN_ID`.

## 7. Подготовка конфигурации

Перейдите в папку проекта:

```bash
cd /opt/ViaVpn
```

Откройте файл конфигурации:

```bash
nano qq.env
```

Отредактируйте `qq.env` в соответствии с вашими данными.

> Не публикуйте этот файл в GitHub. Для публичного репозитория используйте `.env` или храните секреты только на сервере.
